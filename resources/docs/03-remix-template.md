# Remix Template

Ziel der App ist es ohne Remix-Vorlage auszukommen. Das bedeutet jede Datei
wird als Opt-In dann eingeführt, wenn das entsprechende Feature benötigt wird.
Insbesondere hilft das mir persönlich beim Erlernen von Remix.

Das gleiche gilt für jede Zeile in jeder Konfigurationsdatei.

Im Laufe der bisherigen Entwicklung sind aber beide entry-Files im Repository
gelandet:

- `entry.client.ts`: Ohne strict-Mode
- `entry.server.ts`: Environment-Validierung und Clean-Up
